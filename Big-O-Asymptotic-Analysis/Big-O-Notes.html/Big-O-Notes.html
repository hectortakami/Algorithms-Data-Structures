What is good code? 

3 pillars of Programming
  • Readable : Generally clean, others can understand it easily
  • Scalable: It represents the capability of the code to keep a good performance through it increase the amount of inputs 
  • Memory: Space complexity reduced in the usage of memory



Big-O Notation - Scalability
- Tells us how well a problem is solved! Is used to described how efficient is to run a code



  • O ( 1 ) | Constant Time
    • No matter how much the input data increase, we only do one single process
  • O ( n ) | Linear Time
    • We have to iterate through each individual item, growing the run-time as much inputs the function receives ( Simple Loop ).
  • O ( n^2 ) | Quadratic Time
    • We have to iterate n-times through one input and in each of that, iterate another n-times again ( Nested Loops ). 
  • O ( n! ) | Factorial
    • Adding a nested loop for every input we iterate over

Simplifying Big-O 
  1. Care about the worst case scenario
  2. Drop the constants
  3. Different inputs should have different variables. O( a + b ). A and B arrays nested would be O ( a * b )
    1. Add ( + ) for steps in same indentation level
    2. Multiply ( * ) for nested loops in different indentation level
  4. Care about the most dominant notation

Tricky Interview Questions

Q:  Tell me the Big-O complexity in doing “THIS_STRING”.length
A:  It depends! In JavaScript and Python it’s a build-in method that comes with the method assigned as a property to the string, so access to it cost O( 1 ). 
     Otherwise if the method is length( ) with parenthesis as in Java (before JDK 1.6) it is constructed to iterate through each char and retrieve the sum giving a complexity of O( n ).
